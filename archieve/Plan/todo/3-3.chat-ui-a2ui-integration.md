# Checkpoint 04a: Chat UI + A2UI Integration

## 목표
- Chat UI 안에서 A2UI surface를 안정적으로 렌더링하고, 사용자 액션을 다시 에이전트로 round-trip 시킨다.
- 기본 카탈로그 대신 shadcn 기반 DS 어댑터 카탈로그를 강제한다.

## 구현 체크리스트
- [ ] Chat UI 렌더링 모델 정의:
  - [ ] chat message(텍스트)와 A2UI surface(구조 UI) 공존 규칙
  - [ ] 멀티 surface 표시/전환 규칙
- [ ] Chat bootstrap 규칙 정의:
  - [ ] 기본 카탈로그 auto-init 사용 금지 정책
  - [ ] shadcn adapter registry 초기화 순서 고정
- [ ] 이벤트 루프 정의:
  - [ ] userAction -> transport -> agent -> A2UI message -> surface update
  - [ ] 실패/지연 시 fallback(재시도, 오류 배너, 안전한 기본 상태)
- [ ] 데이터 동기화 규칙 정의:
  - [ ] 채팅 컨텍스트와 surface 데이터모델 경계
  - [ ] surface clear/update/delete 타이밍
- [ ] E2E 시나리오 정의:
  - [ ] 텍스트 질의 -> UI 생성
  - [ ] UI 입력/클릭 -> 응답 반영
  - [ ] 오류/타임아웃/부분 업데이트 복구
