# CP1: Branch & Merge Strategy

## 브랜치 규칙
- 기본 규칙: 기능/태스크 브랜치는 `codex/*` 접두사 사용
- Checkpoint 단위 브랜치 예시: `codex/cp1-integration`, `codex/cp1-1-goal` 
- 병렬 태스크는 독립 작업 브랜치 유지(충돌 최소화)

## 리뷰/병합 기준
- PR은 다음이 충족되어야 병합 허용:
  - 해당 작업의 체크리스트 완료
  - 관련 문서 업데이트(산출물 파일)
  - 주요 테스트/검증 결과 첨부
  - 충돌 해결 근거 로그가 남아 있음

## 릴리즈 전환 기준
- 이전 체크포인트 게이트가 통과된 브랜치에서만 다음 체크포인트로 넘어간다.
- checkpoint 통합 브랜치는 각 단계 종료 후 생성된 문서/코드 기준으로만 리셋 없이 연속 적용한다.
